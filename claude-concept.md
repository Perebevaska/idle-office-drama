# CLAUDE.md — Проект: Idle Office Drama (рабочее название)

## Контекст для Claude Code

Ты помогаешь разрабатывать мобильную idle игру — сначала как Telegram Mini App,
потом как нативное Android приложение. Читай этот файл целиком перед любым действием.

---

## Идея игры — одной фразой

> Ты CEO. Сотрудники живут сами. Ты заходишь раз в несколько часов и читаешь
> что они натворили. Принимаешь решения. Уходишь. Мир продолжается без тебя.

---

## Что делает игру уникальной

Офисные idle игры существуют (Idle Office Tycoon, Office Cat, Office Life),
но все они — про числа и апгрейды. Сотрудники там просто множители прибыли.

Наша ниша: **живой нарратив внутри idle механики.**

- Сотрудники имеют характеры, отношения, историю
- События генерируются из комбинации характеров + состояния компании + истории отношений
- Игрок возвращается и читает "что случилось" как эпизод сериала
- Решения влияют на отношения, не только на цифры

---

## Целевая аудитория

- Казуальные игроки, предпочитающие пассивный режим
- Люди которые любят "числа растут" + хотят к чему-то привязаться
- Возраст 18–35, смешанная аудитория
- Заходят в игру 2–4 раза в день по 2–5 минут

---

## Core Loop (главная петля)

```
ЗАХОДИШЬ
  └── читаешь ленту событий (что случилось офлайн)
  └── принимаешь 2-3 решения по ситуациям
  └── нанимаешь / увольняешь / переставляешь сотрудников
УХОДИШЬ
  └── система живёт дальше без тебя
  └── сотрудники взаимодействуют, генерируют новые события
```

## Механики

### Сотрудники
Каждый сотрудник — объект с полями:

```typescript
interface Employee {
  id: string
  name: string
  role: string           // должность
  level: number          // 1–10
  traits: Trait[]        // характер: ['амбициозный', 'ленивый', 'интриган']
  stats: {
    productivity: number // 0–100
    mood: number         // 0–100
    loyalty: number      // 0–100
    stress: number       // 0–100
  }
  relationships: Record<string, number>  // employeeId → [-100, 100]
  history: HistoryEntry[]                // лог прошлых событий
}
```

Характеры (traits) определяют:
- Какие события могут с ним происходить
- Как он реагирует на решения игрока
- Как взаимодействует с другими

### Движок событий — ключевая архитектура

Не хранить готовые тексты. Хранить шаблоны + правила генерации.

```typescript
interface EventTemplate {
  id: string
  template: string           // "{actor} {action} {target} из-за {reason}"
  conditions: {
    actorTraits?: Trait[]    // какие черты нужны у actor
    minStress?: number       // минимальный стресс для триггера
    relationshipBelow?: number // конфликт только если отношения < N
    companyMoodBelow?: number
    // и т.д.
  }
  outcomes: Outcome[]        // что меняется после события
  choices: Choice[]          // варианты решений для игрока
}
```

Пример события:
```typescript
{
  id: "yogurt_conflict",
  template: "{actor} съел йогурт {target} из холодильника. {target} в ярости.",
  conditions: {
    sameдепартment: true,
    relationshipBelow: 20
  },
  outcomes: {
    onIgnore:  { relationship: -10, mood: -5 },
    onResolve: { relationship: +15, cost: 500 },
    onFire:    { removeActor: true, teamMood: -20 }
  },
  choices: [
    { label: "Игнорировать", outcome: "onIgnore" },
    { label: "Купить всем йогурты", outcome: "onResolve" },
    { label: "Уволить виновного", outcome: "onFire" }
  ]
}
```

**Уникальность достигается через:**
- 50+ шаблонов событий
- Подстановку реальных имён и истории персонажей
- Условия: событие A не повторится пока не пройдёт время или не изменится контекст
- Цепочки: событие может породить следующее через N тиков

### Прокачка

**Сотрудники (автоматически):**
```
Ур.1 → просто работает
Ур.3 → появляется характер, начинает влиять на коллег
Ур.5 → mentor/toxic эффект на соседей
Ур.8 → "легенда отдела", сложнее уволить
Ур.10 → переходит в prestige пул как Легендарный сотрудник
```

**CEO (игрок):**
```
Стажёр-CEO   → 3 сотрудника, 1 отдел
Менеджер     → 2 отдела, новые типы решений
Директор     → корпоративная культура, глобальные законы
VP           → пассивный доход без захода 4ч → 8ч
Легенда      → prestige доступен
```

### Prestige петля (эндгейм)

```
Компания достигает потолка (IPO / поглощение)
  → Продаёшь бизнес
  → Лучшие сотрудники становятся "Легендарными" (постоянная коллекция)
  → Начинаешь новую компанию
  → Легендарные дают перманентные бонусы
  → Новый цикл быстрее + новые события открываются
```

Цель игрока: собрать команду легенд через несколько циклов.


Дополнительно — Telegram Stars за косметику (темы офиса, имена).

## Технический стек

### MVP (Telegram Mini App)
```
Frontend:   React + TypeScript + Tailwind CSS
Логика:     чистый TypeScript (платформо-независимый core)
Хранение:   localStorage (MVP) → Supabase (после валидации)
Платформа:  Telegram Mini App API (@twa-dev/sdk)
Деплой:     Vercel
```

### Важнейший архитектурный принцип
Core логика игры (движок событий, сотрудники, прокачка) —
**полностью отдельный TypeScript модуль** без зависимостей от Telegram или React.

```
src/
  core/              ← платформо-независимая логика
    engine.ts        ← главный игровой цикл
    employees.ts     ← модель сотрудников
    events.ts        ← движок событий
    progression.ts   ← прокачка и prestige
    templates/       ← шаблоны событий (JSON)
  
  telegram/          ← обёртка для Telegram Mini App
    auth.ts
    payments.ts
    share.ts
  
  ui/                ← React компоненты
    screens/
    components/
  
  store/             ← состояние (Zustand или Redux)
```

Это позволит потом перенести core в React Native / Unity без переписывания логики.
